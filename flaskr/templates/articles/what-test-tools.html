Break down each tool. Analyse the best situations to use it for

Jmeter: Java, GUI driven, average recorder, nice plugins, reasonable reporting, completely free

k6: JS based, cmd, API testing focussed, no distributed testing mode,

Loadrunner: C based, GUI driven, good recorder, firewall handling, expensive

A tool is built to solve for a particular problem. Performance test tools are no different. Each one needs to be selected to fit the most appropriate use case. Some tools are good enough where it can resolve 90% of scenarios that you'll come across.

Main features to look for are time to get setup, flexibility in transactions, costs. Everything after comes down to preference in the same way some have a preference for firefox over chrome. They both do the same job, but in a different way
