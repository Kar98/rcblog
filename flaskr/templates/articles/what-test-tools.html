How to choose the right tool

The first question you need to ask is, what problem am I trying to solve?

A tool is built to solve for a particular problem and performance test tools are no different. A hammer is built to hit things, a saw to saw through things. You can use a saw to hammer in nails but it's going to take longer and have a number of limitations.

Once you understand what your problem is then you can look at the core features each tool needs to solve. There will be differences between them but from my experience all performance test tools need to deal with the following concerns:

  • Time to get setup
  • Flexibility in creating a user flow
  • Costs

Brief overview of the key tools to use

Jmeter: Java, GUI driven, average recorder, nice plugins, reasonable reporting, completely free

k6: JS based, cmd, API testing focussed, no distributed testing mode,

Loadrunner: C based, GUI driven, good recorder, firewall handling, expensive
